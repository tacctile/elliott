# Production Process

## Process Steps

1. **File prep** — import 1277970 print/cut file, set up registration marks, define kiss-cut path. Estimated 15-20 min per unique design (4 designs in this program → ~60-80 min total file prep across the program).
2. **Print** — Roland SG3-300, full-color eco-solvent on Orajet 3951. All 4 designs nested in a single small print area; one print pass.
3. **Laminate** — 1-mil clear polyester overlay, single pass through the 13.5" laminator (the entire sheet of 20 labels easily fits).
4. **Kiss cut** — Roland contour cut, face stock only, liner intact. Slit liner across center per drawing spec.
5. **Inspect** — print quality, laminate adhesion, kiss-cut depth, text legibility (small format — readability check is critical).
6. **Package** — flat, on liner, ready for hand application on the control panel.

# Pricing Derivation

**Methodology:** Minimum run / job economics. Built from scratch as the founding data point for the sub-0.1 sq ft printed/laminated size class.

**Why the sq ft band is inapplicable:**

| Approach | Result | Verdict |
|----------|--------|---------|
| Apply Printed + Laminated singles band ($15.43/sq ft at qty 20) to this item | 0.008 sq ft × $15.43 = **$0.12/label** | Nonsensical — does not cover setup. |
| Apply Printed + Laminated kit band ($16.42/sq ft at qty 20) | 0.008 sq ft × $16.42 = **$0.13/label** | Same problem. Not a kit anyway. |
| Minimum run / job economics | Setup + file prep + print + lam + kiss-cut + inspect + package dominate cost. | **Adopted.** |

**Why these labels don't price on sq ft:** The existing Pricing Profile band was derived from items 150× larger by area (1.296 sq ft and up). At those sizes, material consumption and run-time scale with area in a way that supports per-sq-ft pricing. At 0.008 sq ft, material cost is fractions of a cent and run-time is trivial. The fixed costs — file prep per unique design, color profiling, print registration, kiss-cut definition, lamination setup, inspection, packaging — are the same as for any small printed label job, regardless of label size. Therefore the price reflects what it costs to *bother doing the job at all*, not the material or area consumed.

**Job economics anchor (one-off, qty 5 per design):**

| Component | Estimate |
|-----------|----------|
| File prep + cut path definition (per unique design) | ~$20-30 |
| Print + lam + kiss-cut + inspect + package (program total, 20 labels) | ~$50-70 |
| Total job floor (4 designs × qty 5) | ~$130-170 |
| Per label across the program | ~$6.50-8.50 |

Adopted **$7.00/label at qty 1-9**. Total program: 4 × 5 × $7 = **$140**. Lands within the job-floor estimate range.

**Tier compression** (1-9 → 200+ = 79% discount): steeper than the standard printed/laminated singles compression (63%). Justified because variable per-label cost is near-zero at this size, so volume amortizes the fixed setup over many more units.

**Profile Validation:** This item is **outside** the Printed + Laminated singles Pricing Profile band by design and by structural necessity. It establishes a *new* size class within the Orajet 3951 family — tiny printed labels under 0.1 sq ft — that prices on job economics rather than sq ft. Do NOT contaminate the existing singles band with this data point.

**AI Validation:** Not run. Per the session brief, multi-round AI validation is not warranted for this one-off — the pricing logic is straightforward (job economics, same material family as existing catalog) and Sean has explicitly stated no reorder expectation. Profile band validation (the structurally relevant check) was performed and documented above.

**Override:** None.

**Strategic note:** This is the first quoted item in the sub-0.1 sq ft printed label size class on this account. If similar small printed labels appear in future Elliott programs, this item is the precedent — but the price was set for a one-off, so future structurally similar items at higher volumes may warrant a fresh cost-build.
